# Auto-Formatting Indent Bugs - 2026-03-17

> Ultimate Boss Script (`test/oracle_format_ultimate_boss.sql`) 검증에서 발견된 indent 버그 목록

## BUG 5 (심각): `EXTRACT(YEAR FROM ...)` - `FROM`이 SQL 절 키워드로 오인

`EXTRACT(YEAR FROM ...)`, `EXTRACT(MONTH FROM ...)`, `EXTRACT(DAY FROM ...)` 내부의 `FROM`이
SQL `FROM` 절로 인식되어 indent level 0으로 줄바꿈. SELECT 리스트 전체 indent 구조가 파괴됨.

```sql
-- 원본
SELECT
    EXTRACT(YEAR FROM d.dt)  AS yyyy,
    EXTRACT(MONTH FROM d.dt) AS mm,
    EXTRACT(DAY FROM d.dt)   AS dd

-- 포맷팅 결과 (깨짐)
SELECT d.dt,
    EXTRACT (YEAR
FROM d.dt) AS yyyy,
    EXTRACT (MONTH
FROM d.dt) AS mm,
    EXTRACT (DAY
FROM d.dt) AS dd
```

- 위치: `format_statement()` 내 clause keyword 분기
- 원인: `FROM`이 `FORMAT_CLAUSE_KEYWORDS`에 포함되어 있고, `EXTRACT(` 내부 context를 구분하지 않음
- 영향: `EXTRACT FROM`을 사용하는 모든 쿼리에서 SELECT 리스트 indent 붕괴

## BUG 6: `RETURNING` inside JSON 함수가 절 키워드로 오인

`JSON_VALUE`, `JSON_OBJECT`, `JSON_ARRAYAGG` 내부의 `RETURNING CLOB` / `RETURNING VARCHAR2(...)` 가
SQL `RETURNING` 절로 인식되어 불필요한 줄바꿈 발생.

```sql
-- 원본
JSON_VALUE(e.json_profile, '$.level' RETURNING VARCHAR2(30)) AS profile_level

-- 포맷팅 결과 (깨짐)
JSON_VALUE (e.json_profile, '$.level'
    RETURNING VARCHAR2 (30)) AS profile_level
```

```sql
-- 원본
JSON_OBJECT(... RETURNING CLOB)

-- 포맷팅 결과 (깨짐)
JSON_OBJECT (...)
RETURNING CLOB)
```

- 위치: `format_statement()` 내 clause keyword 분기
- 원인: `RETURNING`이 `FORMAT_CLAUSE_KEYWORDS`에 포함되어 있고, 함수 괄호 안인지 구분 없음
- 영향: JSON 함수를 사용하는 쿼리에서 indent 깨짐 + 괄호 매칭 시각적 혼란

## BUG 7: MATCH_RECOGNIZE 내부 하위절 미인식

MATCH_RECOGNIZE 블록 내부의 고유 키워드(`MEASURES`, `PATTERN`, `DEFINE`, `ALL ROWS PER MATCH` 등)가
절 분리되지 않아 한 줄로 뭉침.

```sql
-- 원본
MATCH_RECOGNIZE (
    PARTITION BY emp_id
    ORDER BY sale_date, sale_id
    MEASURES
        MATCH_NUMBER()  AS match_no,
        CLASSIFIER()    AS cls,
        FIRST(sale_date) AS first_sale_date
    ALL ROWS PER MATCH
    PATTERN (low+ mid* high)
    DEFINE
        low  AS (...) < 10000,
        mid  AS (...) BETWEEN 10000 AND 30000,
        high AS (...) > 30000
)

-- 포맷팅 결과 (뭉침)
MATCH_RECOGNIZE (
    ...
    ORDER BY sale_date,
        sale_id MEASURES MATCH_NUMBER () AS match_no,
        ... ALL ROWS PER MATCH PATTERN (low + mid * high) DEFINE low AS ...,
        mid AS ...,
        high AS ...
)
```

- 위치: `format_statement()` - MATCH_RECOGNIZE 전용 하위절 처리 부재
- 원인: `MEASURES`, `PATTERN`, `DEFINE` 등이 clause keyword로 등록되어 있지 않음
- 영향: MATCH_RECOGNIZE 블록 가독성 소실

## BUG 8: MODEL 하위절 미인식

MODEL 절 이후의 `PARTITION BY`, `DIMENSION BY`, `MEASURES`, `RULES` 등 하위 키워드가
절 분리되지 않아 한 줄로 연결됨.

```sql
-- 원본
MODEL
    PARTITION BY (year_key, channel_code)
    DIMENSION BY (month_key)
    MEASURES (base_amt, proj_amt)
    RULES SEQUENTIAL ORDER (...)

-- 포맷팅 결과 (한 줄)
MODEL PARTITION BY (...) DIMENSION BY (...) MEASURES (...) RULES SEQUENTIAL ORDER (...)
```

- 위치: `format_statement()` - MODEL 전용 하위절 처리 부재
- 원인: MODEL 고유 키워드가 clause keyword로 등록되어 있지 않음
- 영향: MODEL 절 가독성 소실

---

## BUG 9: PIVOT / UNPIVOT 내용 한 줄 collapse

PIVOT, UNPIVOT 블록 내부 내용이 모두 한 줄로 압축됨.

```sql
-- 원본
PIVOT (
    SUM(net_amt)
    FOR channel_code IN (
        'ONLINE'  AS online_amt,
        'DIRECT'  AS direct_amt,
        'PARTNER' AS partner_amt
    )
)

-- 포맷팅 결과 (한 줄)
PIVOT (SUM (net_amt) FOR channel_code IN ('ONLINE' AS online_amt, 'DIRECT' AS direct_amt, 'PARTNER' AS partner_amt))
```

- 위치: `format_statement()` - `(` 처리 시 PIVOT/UNPIVOT context 미구분
- 원인: PIVOT/UNPIVOT 뒤의 `(`가 subquery로 인식되지 않아 `suppress_comma_break_depth` 증가
- 영향: PIVOT/UNPIVOT 블록 가독성 소실
