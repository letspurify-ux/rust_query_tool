# DB 세션 관리 원칙

이 문서는 쿼리 탭과 DB 접속 세션의 소유권, 종료, 정리 원칙을 기록한다.

## 전제

- 세션은 쿼리 실행에 사용하는 DB connection 또는 transaction context를 뜻한다.
- 세션이 생성된 뒤에는 해당 쿼리 탭이 세션의 소유자다.

## 원칙

- 세션은 탭별로 유지한다.
- 세션은 탭에서 첫 쿼리를 실행할 때 생성한다.
- `commit`, `rollback`은 명령을 요청한 순간 선택된 탭의 세션에 적용한다.
- `cancel`은 모든 쿼리 탭의 실행 중인 세션과 lazy fetch 세션에 적용한다.
- 쿼리 실행 중인 탭을 닫으려는 경우, 사용자가 다음 중 하나를 선택해야 한다.
  - 실행 중인 쿼리를 취소하고 탭을 닫는다.
  - 탭 닫기를 취소한다.
- 커밋이나 롤백이 필요한 탭을 닫으려는 경우, 사용자가 다음 중 하나를 선택해야 한다.
  - 커밋하고 탭을 닫는다.
  - 롤백하고 탭을 닫는다.
  - 탭 닫기를 취소한다.
- 탭이 닫히면 관련 세션은 명시적으로 종료한다.
- 소유 탭이 없는 세션은 고아 세션으로 보고 정리한다.

## 구현 메모

- `commit`, `rollback` 대상 탭은 사용자가 명령을 요청한 시점에 고정해야 한다.
  이후 선택 탭이 바뀌어도 이미 요청된 명령의 대상 세션이 바뀌면 안 된다.
- `cancel`은 명령 요청 시점에 실행 중이거나 lazy fetch 중인 모든 탭의 세션을 대상으로 한다.
- 커밋이나 롤백이 필요한지는 가능하면 DB 또는 드라이버가 제공하는 transaction 상태를 기준으로 판단한다.
- transaction 상태를 신뢰성 있게 확인할 수 없다면, 미커밋 작업이 있을 수 있는 세션을 조용히 닫지 않도록 보수적인 dirty-session 신호를 사용한다.
- 고아 세션 정리는 탭 닫기 시점에 수행하고, UI 상태 불일치나 예외 상황에 대비해 주기적인 cleanup으로도 수행할 수 있다.
