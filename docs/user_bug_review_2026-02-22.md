# 일반 유저 기능 버그 리뷰 20건 (2026-02-22)

1. 연결 표시 문자열이 `username@@host` 형태로 렌더링되어 사용자에게 잘못된 접속 정보가 표시됩니다.
2. 최근 연결 저장 시 keyring 저장 실패에도 비밀번호를 즉시 제거하여 재연결 시 비밀번호 유실이 발생할 수 있습니다.
3. 레거시 비밀번호 마이그레이션 중 keyring 저장 실패 이후에도 비밀번호를 지워 복구 불가 상태를 만들 수 있습니다.
4. 설정 파일 경로를 구하지 못한 경우 저장을 건너뛰면서도 성공(`Ok(())`)으로 처리해 silent failure가 납니다.
5. 히스토리 저장도 경로 부재 시 성공으로 처리되어 사용자는 저장되었다고 오인할 수 있습니다.
6. 히스토리 저장 임시 파일명이 고정(`json.tmp`)이라 다중 인스턴스 실행 시 충돌 위험이 있습니다.
7. 저장된 연결 더블클릭 시 `Connect(info, true)`로 전달되어 사용자 의도와 무관하게 저장 플래그가 강제됩니다.
8. keyring 비밀번호 조회 실패를 빈 문자열로 대체하여 즉시 연결 시 인증 실패를 유발할 수 있습니다.
9. 히스토리 writer 채널 장애 fallback에서 `history.save()` 오류를 버려 데이터 저장 실패가 UI로 전달되지 않습니다.
10. 히스토리 flush 대기시간이 1.5초 고정이라 느린 디스크 환경에서 정상 작업을 timeout 실패로 오판할 수 있습니다.
11. 오류 라인 추출이 텍스트 내 첫 번째 `line` 숫자만 읽어 실제 SQL 오류 라인과 불일치할 수 있습니다.
12. `IDENTIFIED BY` 마스킹이 SQL 주석/문자열 문맥을 고려하지 않아 원문이 왜곡될 수 있습니다.
13. URI 마스킹 시 `userinfo`에 `:`가 없으면 그대로 노출하여 일부 자격증명 패턴을 가리지 못합니다.
14. CONNECT 문 마스킹 fallback이 `rfind('@')` 기반이라 비밀번호에 `@`가 포함되면 경계 인식이 깨질 수 있습니다.
15. 히스토리 목록 snapshot timeout 시 디스크 fallback을 사용해 최신 메모리 상태가 누락된 stale 목록을 보여줄 수 있습니다.
16. 연결 테스트 버튼 연타 방어가 없어 클릭마다 테스트 스레드가 생성되어 네트워크/DB에 과부하를 줄 수 있습니다.
17. 연결 삭제 중 keyring 삭제 실패를 stderr 로그만 남기고 UI에 실패 상태를 표시하지 않습니다.
18. 연결 정보 생성 시 host/service에 대한 형식 검증이 거의 없어 잘못된 값이 저장·재사용됩니다.
19. QueryHistory 로드 중 레거시 마이그레이션 저장 실패(`let _ = history.save()`)를 무시해 실패 탐지가 어렵습니다.
20. 최근 연결 개수를 `truncate(10)`으로 고정해 오래된 연결이 사용자 동의 없이 자동 제거됩니다.
