# 일반 유저 기능 버그 정밀 점검 20건 (2026-02-22, round3)

아래 항목은 일반 사용자가 직접 체감하는 기능(연결/기록/검색/로그/설정) 중심으로 코드 정적 리뷰에서 식별한 **잠재 버그 20건**입니다.

1. **히스토리 스냅샷 수신 타임아웃 시 디스크 fallback으로 최신 상태 유실 가능**  
   `load_snapshot()`는 writer 스레드 응답이 지연되면 파일 기반으로 즉시 fallback 하므로, 아직 flush되지 않은 최신 쿼리가 누락될 수 있습니다.

2. **히스토리 flush timeout이 단일 타임아웃 실패로 바로 에러 처리됨**  
   로그 writer는 재시도 경로가 있지만, 히스토리 writer flush는 timeout 시 즉시 실패 처리되어 느린 환경에서 false negative 위험이 있습니다.

3. **히스토리 검색이 locale-aware가 아닌 단순 `to_lowercase` 기반**  
   다국어(터키어 등) 케이스 폴딩에서 기대 검색 결과와 불일치가 발생할 수 있습니다.

4. **Find/Replace 대소문자 무시 검색이 ASCII 전용 매칭**  
   `eq_ignore_ascii_case`로 byte 매칭하기 때문에 비ASCII 문자의 case-insensitive 검색 정확도가 낮습니다.

5. **Replace 단건 비교는 Unicode `to_lowercase`, 전체 ReplaceAll은 ASCII 매칭으로 규칙 불일치**  
   같은 "대소문자 무시" 옵션인데 동작 경로별 매칭 기준이 달라 결과가 달라질 수 있습니다.

6. **로그 뷰어 메시지 절단에서 최종 길이 보장 불명확**  
   문자열 절단 후 `...`를 덧붙이는 방식이라 표시 최대 길이에 대한 사용자의 기대와 실제 길이가 어긋날 수 있습니다.

7. **히스토리 SQL 절단도 동일하게 표시 길이 체감 불일치 가능**  
   문자 수 기준 절단 + `...` 부착 정책으로 열 폭 고정 UI에서 기대 폭과 실제 표시 폭이 달라질 수 있습니다.

8. **연결 삭제 시 keyring 삭제 실패가 나도 UI 목록은 삭제된 것처럼 갱신됨**  
   `remove_connection()`이 keyring 에러를 반환해도 목록은 이미 `retain`으로 제거되어, 사용자 입장에서 부분 실패 상태를 혼동할 수 있습니다.

9. **연결 삭제 후 config 저장 실패 시 메모리 상태는 이미 변경됨**  
   저장 실패 alert 후에도 현재 세션 메모리에는 삭제 반영 상태가 남아 UX 불일치가 생길 수 있습니다.

10. **연결 테스트 동시 실행 방지가 버튼 비활성화 타이밍 의존**  
    callback 진입 직후 비활성화/메시지 전송 구조라 매우 빠른 연타나 이벤트 적체 상황에서 중복 테스트 요청 누적 여지가 있습니다.

11. **연결 테스트 스레드가 종료된 dialog 수명과 분리되어 결과 메시지 유실 가능**  
    다이얼로그 종료 직후 백그라운드 테스트 결과가 도착하면 사용자에게 결과 피드백이 사라질 수 있습니다.

12. **host 유효성 검증이 국제화 도메인/특수 케이스를 과도하게 배제**  
    ASCII 알파뉴메릭 + 일부 기호만 허용해, 실제 환경에서 유효한 호스트 표기가 거부될 수 있습니다.

13. **service name 검증 규칙이 실제 DB 네이밍 규칙과 불일치 가능**  
    허용 문자셋이 고정되어 있어 환경별 service 식별자 입력이 차단될 여지가 있습니다.

14. **URI credential 마스킹에서 마지막 `@` 기준 파싱 의존**  
    userinfo 내부 `@` 포함 등 복합 케이스에서 경계 인식이 어긋나 과/소 마스킹이 날 수 있습니다.

15. **URI credential 마스킹 종료 문자셋이 제한적이라 URL 변형 케이스 누락 가능**  
    authority 종료 판정이 특정 문자 집합 기반이라 비정형 URI에서 마스킹 범위 오류 가능성이 있습니다.

16. **에러 라인 파싱이 문자열 패턴 하드코딩 의존**  
    드라이버/버전별 메시지 포맷 차이를 충분히 흡수하지 못해 잘못된 라인 하이라이트가 발생할 수 있습니다.

17. **에러 라인 파싱이 숫자 범위 유효성(0행 등)을 검증하지 않음**  
    파싱된 line 값이 실제 SQL 라인 범위를 벗어나도 보정 없이 전달될 수 있습니다.

18. **폰트 검색 필터가 locale-aware 정규화 없이 단순 소문자 포함 비교**  
    사용자 로케일/문자 정규화 조합에 따라 폰트 검색 결과가 직관과 다를 수 있습니다.

19. **로그 저장 경로(`dirs::data_dir`)가 없으면 로그 저장 불가 상황이 빈번히 사용자에게 노출될 수 있음**  
    일부 실행 환경(샌드박스/포터블)에서 경로 탐색 실패 시 로그 기능 전체가 사실상 비활성화됩니다.

20. **Crash log 쓰기 실패가 모두 무시되어 장애 분석 단서 유실 가능**  
    panic 훅 경로에서 디렉터리 생성/쓰기 오류를 모두 무시(`let _ = ...`)해, 실제 장애 시 원인 추적 자료가 사라질 수 있습니다.
